A GitOps pipeline where Git is the only source of truth.

Terraform provisions the AWS/EKS infrastructure. Jenkins builds, tests, and pushes the image. ArgoCD watches the repo and syncs the cluster to match it — no kubectl apply ever runs from CI.

01

Architecture

terraform/
Infrastructure as Code. VPC, EKS cluster, managed node group, and IAM roles — provisioned with remote state (S3 + DynamoDB locking).
app/
Sample Flask service with a Dockerfile — the workload the pipeline ships end to end.
helm-chart/
Packaged Kubernetes manifests. Deployment, Service, and HorizontalPodAutoscaler, parameterized through values.yaml.
argocd/
The GitOps contract. An Application manifest with automated sync + self-heal — the cluster is not allowed to drift from Git.
jenkins/
Build stage only. Jenkins builds and pushes the image, then commits a version bump — it never talks to the cluster directly.

Setup order

Bootstrap remote state
Create the S3 bucket + DynamoDB table Terraform needs, terraform init won't run without them.
Provision infrastructure
terraform apply — creates the VPC, EKS cluster, and node group (~15 min).
Build & push the image
Manual first push to Docker Hub, then Jenkins takes over on every commit.
Install ArgoCD
Deploy ArgoCD into the cluster and grab the initial admin password.
Point ArgoCD at this repo
Apply the Application manifest — from here, every commit to helm-chart/ auto-deploys.
